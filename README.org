Люди жаловались, что у livelib нет API. Поэтому в какой-то момент мне
стало интересно можно ли выковырять его из android-клиента. 

Подключил телефон через ноут к интернету и стал смотреть wireshark что
там ходит между приложением и сервером. Вполне себе ходят запросы, на
примеры можно посмотреть в файле =livelib_api.org=. Вытащив сессионный
ключ, позапускал запросы к API из браузера.

Занятие это мне быстро надоело, потому что медленно и печально. Взял
apkextractor, вытащил из телефона apk и расковырял его с помощью
apkstudio. Потом просто грепал по исходникам GET- и POST-запросы из
чего получился =api_extract.org=. Там есть все запросы, которые на тот
момент поддерживало приложение.

Аунтентификация сделана через упаковку пароля публичным ключом. Можно
логиниться и получать сессионый ключ. Дальше только хардворчить над
реализацией API и сериализацией данных.

После пары дней экспериментов, мне в браузер прилетела кука
=iwatchyou=. Предполагаю, что надо было просто внимательнее
проставлять =user-agent= в браузере. 

Рядом лежит скрипт =msgs.rb= которым из полученного json c сообщениями
пользователя можно генерить html.
